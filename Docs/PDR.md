# PRD — VibeFeed

## 1. 서비스 목적 (Why)
- 핵심 문제: WIV 빌더톤 참가자들이 서로의 진행 상황을 모르고 고립되어 작업함
- 불편: 카카오톡/슬랙에 올리면 묻히고, 프로젝트 기록이 체계적으로 남지 않음

## 2. 타겟 사용자 (Who)
- 주요 사용자: WIV 빌더톤 참가자 (비개발자 여성)
- IT 숙련도: 초급 (스마트폰 + 기본 PC 사용)
- 사용 환경: 모바일 우선 (PC도 지원)

## 3. 핵심 기능 (What)
1. 이미지 + 텍스트로 게시글 작성 (프로젝트 근황 공유)
2. 피드 형태로 최신 게시글 스크롤 조회
3. 좋아요(하트) 반응
4. 카테고리 태그 필터 (#오늘의배움 #프로젝트진행 #고민상담 #자유)
5. 프로필 페이지 (내 게시글 모아보기)

## 4. 화면 구성 (Pages)
- 페이지 1: 피드 (메인) — 전체 게시글을 최신순으로 스크롤
- 페이지 2: 글쓰기 — 새 게시글 작성 폼
- 페이지 3: 프로필 — 내 정보 + 내 게시글 모아보기

## 5. 디자인 요건
- 전체 톤앤매너: 따뜻하고 여성 친화적, 인스타그램과 유사한 카드형 레이아웃
- 주요 색상: 코랄 핑크(#FF6B6B) + 화이트(#FFFFFF) + 라이트 그레이(#F8F9FA)
- 모바일 대응 여부: 예 (모바일 우선 디자인)
- 참고: 인스타그램의 피드 구조, Pinterest의 카드 레이아웃

## 6. 성공 기준
- [ ] 피드에서 게시글 카드가 이미지+텍스트+좋아요와 함께 보인다
- [ ] 새 글 작성 폼에서 제목·내용·카테고리를 입력하고 등록할 수 있다
- [ ] 카테고리 태그를 클릭하면 해당 태그 게시글만 필터링된다
